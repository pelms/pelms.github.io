'Save As…' problems in Netscape 4.05

Netscape version 4.05 (as currently used in Bolland library) has a few problems with the way it saves web pages. The end result is that you may end up with a saved file which is a web document complete with all the HTML tags when what you thought you were getting was a text file.

The confusion arises from the 'Save as type:' drop-down list in the 'Save As...' dialog box, which has no effect on the format the file is saved in. The only effect this selection has is to change the type of files listed in the folder or disk you are saving to.

What happens:

  1. Netscape will save a web page in the format you specify as the filename extension:
    i.e. .txt will give you a Text file, .htm an HTML file.
    This is true whatever is in the 'Save as Type' box !

  2. If you don't add an extension the file will be saved as an HTML file, with an .htm extension. Again, this happens whatever you set in the 'Save as Type' box.

  3. If you give the page an extension other than 'txt' or 'htm' it will be saved as an HTML file but with the extension you give it.

In summary:

Ignore the 'Save as type' field; it does not do anything in this version of Netscape!

To save a web page as a text file (i.e. stripped of all the HTML tags), add a 'txt' extension.

To save a web page as an HTML file, add an 'htm' extension or no extension at all to the filename.

Netscape version 4.5 (as on the NT PCs)

In this version of Netscape the 'Save as type' list box does have a use: If you save a web page with a filename with no extension, it will be saved to the format selected in this list.

Internet Explorer 5.0 (as on the NT PCs)

This is much more logical:
The page is saved in whichever format you specify in the 'Save as type' list box, whatever extension you give it. If you don't add an extension the appropriate one will be added. If you add another extension besides 'txt', 'htm' or 'html' this extension will be used and the file saved in the format in the the 'Save as type' list box.
